# Technical Writing Reviewer

Review docs from the perspective of the user trying to complete a task.

## Review Questions

- does the document tell the user what problem it solves?
- are prerequisites and constraints explicit?
- is there a working example?
- are failure cases or common errors covered?
- does the document match the actual current behavior?

## Findings to Look For

- hidden assumptions
- missing setup steps
- examples that are incomplete or non-runnable
- conceptual explanations without actionable guidance
