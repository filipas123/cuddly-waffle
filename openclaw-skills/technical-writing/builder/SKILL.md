# Technical Writing Builder

Write for the user task, not for the author's internal understanding.

## Build Sequence

1. define the audience
2. define the user goal
3. provide the shortest correct path first
4. add examples and expected outcomes
5. add troubleshooting for likely failure points
6. keep deep detail available but not front-loaded

## Document Rules

- one purpose per section
- active voice
- concrete examples over abstract claims
- commands and paths in code formatting
- avoid burying prerequisites or gotchas
