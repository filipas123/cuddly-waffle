# Optimization Builder

Optimize the bottleneck you actually have.

## Build Sequence

1. Measure latency, token use, and quality.
2. Identify the dominant bottleneck.
3. Apply one class of optimization at a time.
4. compare before and after.
5. keep rollback simple.

## Optimization Levers

- prompt compression
- context pruning
- retrieval budget reduction
- model routing by difficulty
- output schema tightening
- caching repeated work
- streaming partial results

## Guardrail

An optimization that saves cost but damages core answer quality is not a win unless the tradeoff is explicit and accepted.
