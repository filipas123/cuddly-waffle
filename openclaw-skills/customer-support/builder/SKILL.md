# Customer Support Builder

Build support behavior around clarity, speed, and safe escalation.

## Build Sequence

1. identify the issue class
2. define required diagnostic inputs
3. define first response pattern
4. define workaround path if available
5. define escalation criteria
6. define closure conditions

## Support Rules

- ask only for information that changes the next action
- separate explanation from next step
- do not pretend certainty when diagnosis is incomplete
- preserve user context across follow-ups when appropriate
